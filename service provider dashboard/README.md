# 🚨 ResQTrack – Command Center for Roadside Emergency Assistance

**ResQTrack** is a modern, scalable platform built to revolutionize roadside emergency services. With real-time tracking, AI-powered assistance, automated request handling, and a seamless UI, this command center serves as the core dashboard for managing and coordinating vehicle emergencies.

---

## 🌐 Overview

ResQTrack bridges the gap between stranded vehicle users and roadside service providers. This project is the _Command Center_, designed specifically for emergency response operators, mechanics, and backend admins to view, manage, and respond to incoming vehicle emergency requests efficiently.

---

## ⚙️ Tech Stack

- **Frontend:** React + TypeScript (Vite)
- **UI:** Tailwind CSS + ShadCN UI
- **Routing & State:** React Router DOM, Zustand/Context API
- **Other Tools:** Lovable for rapid iteration, GitHub for version control

---

## 🚀 Features

### 🧭 Real-Time Monitoring

- Track vehicle emergency requests across different locations
- Live dashboard with request logs and statuses

### 🆘 Emergency Request Management

- View all active and historical emergency cases
- Filter, sort, and search requests based on urgency, location, or status

### 🔍 Request Detail Dialog

- View full metadata of service requests in a modal dialog
- Assigned mechanic, vehicle issue, priority, contact, etc.

### 👥 Service Provider Dashboard

- Accept/Reject service requests
- Track task completion and performance
- Emergency alert support system for instant communication

### 🤖 AI-Powered ResQBot (Integrated in Mobile App)

- Voice-based emergency reports
- Auto-triggered emergency alerts via QR Code (ResQTag)

### 📄 Auto Logging & Receipt Generation

- Each interaction is logged
- Generate and download service invoices

### 📌 ResQTag System (Backend Enabled)

- QR Code placed on vehicles
- Scanning sends auto-alerts with exact location to family, hospitals & command center
